# Photo Stream

Photo Stream is a simple, self-hosted Jekyll photo stream. It publishes a static photo gallery from images placed in `photos/original`, generates resized image variants, and serves photo pages with clean URLs.

This repository is a maintained fork of the original Photo Stream project. The main purpose of this fork is to keep the project usable with current container tooling and safer dependency defaults while preserving the original user experience.

## Existing Features

- Static, self-hosted photo stream generated by Jekyll
- Automatic photo pages with slug-based URLs
- Lazy-loaded photo grid using tint placeholders
- Generated large, thumbnail, and tint image variants
- Light and dark themes via `prefers-color-scheme`
- Keyboard and swipe navigation for photo detail views
- Optional RSS feed link and Atom feed output
- Optional image sharing controls
- Optional original-image download controls
- Optional footer links for GitHub, Instagram, Twitter, and a custom URL
- Optional header bar and configurable photo sort order

## Updates In This Fork

- Modernized Docker support with a multi-stage image
- Runtime container now runs as an unprivileged user
- Build tools are excluded from the final runtime image
- Docker Compose adds `no-new-privileges` and drops Linux capabilities
- Runtime configuration is documented in `.env.example`
- `.env` is ignored so local/site-specific settings are not committed
- Ruby dependencies were updated for Ruby 3.4 compatibility
- `rexml` was updated to address a known malformed XML denial-of-service advisory
- A GitHub Actions workflow builds and publishes the container image to GHCR
- Small JavaScript and EXIF-processing fixes were applied

## Container Image

This fork publishes a container image to GitHub Container Registry:

```sh
docker pull ghcr.io/doubtfulturnip/photo-stream:latest
```

Published tags include `latest`, branch names, Git tags, and commit SHA tags.

## Basic Usage

1. Copy `.env.example` to `.env`.
2. Edit `.env` for your site title, public URL, author details, and feature flags.
3. Add original photos to `photos/original`.
4. Start the app with Docker Compose:

```sh
docker compose up -d
```

Open the local site at:

```text
http://localhost:4000
```

The compose file builds this fork's local Dockerfile by default. If you want to use the published GHCR image instead, replace the local `build:` block in `docker-compose.yml` with:

```yaml
image: ghcr.io/doubtfulturnip/photo-stream:latest
```

## Photo Output

During a build, images from `photos/original` are processed into:

- `photos/large`
- `photos/thumbnail`
- `photos/tint`

Generated static output is written to `_site`.

By default, the container serves the existing `_site` immediately and runs Jekyll generation in the background when `_site/index.html` already exists. If `_site/index.html` is missing, it runs one foreground build first.

Set `PHOTO_STREAM_SERVE_MODE=static` or `PHOTO_STREAM_BUILD_ON_START=0` to skip Jekyll generation on container restart when `_site/index.html` already exists. Set `PHOTO_STREAM_SERVE_MODE=jekyll` to block startup until generation finishes. Set `PHOTO_STREAM_INCREMENTAL=1` to pass `--incremental` to Jekyll builds.

## Security Notes

This project processes image files through EXIF and image-processing libraries. Only trusted users should be able to add files to `photos/original`.

For public hosting, the safest deployment pattern is to serve the generated `_site` directory from a normal static web server or CDN.
